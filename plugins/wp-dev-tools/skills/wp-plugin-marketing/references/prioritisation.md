# Step 4 — Prioritisation system

Once the foundations exist (MVC assets, .org page, website), the recurring
question is "what do I do next, and is it worth it?". Give the user a system, not
ad-hoc effort. Use this when they ask how to prioritise, what to measure, or how
to keep marketing going consistently.

## The marketing pyramid: vision → goals → objectives → KPIs → data

Work top-down. Each layer constrains the one below, so effort stays aligned.

1. **Vision** — *why* the business exists (the durable "why"). One sentence.
2. **Goals** — broad outcomes the business wants from its channels. Examples:
   increase profit, grow brand awareness, enter a new marketplace, reach a new
   audience.
3. **Objectives** — concrete tasks that move toward the goals (not yet
   numbers). Examples: publish regular blog posts; make a walkthrough video for
   every feature; open accounts on the relevant platforms.
4. **KPIs** — the metrics that measure progress on the objectives. Examples:
   downloads, conversion rate, search traffic, email signups, Customer Lifetime
   Value (CLV). Track over time, because goals shift.
5. **Data** — the source of the KPIs (below).

When a user asks for "a marketing plan", produce this filled in for their
product — vision and goals drafted (marked as assumptions where needed),
objectives concrete, 3–5 KPIs chosen, and the data sources named.

## Data audit — pull everything into one place

There's no single tool that measures it all, so collect into one sheet and read
it together. Typical sources:

**From WordPress.org:** referral traffic to your site, downloads, active
installs, number of reviews, average rating, support tickets (received / closed
/ open).

**From your website:** sessions, users, conversion rate to download/purchase,
best-performing landing pages, top exit pages.

**From email:** number of contacts (by lifecycle stage if available), conversion
rate from subscriber → download/purchase.

In-plugin analytics/usage tracking (e.g. install/deactivation tracking, opt-in
usage data) can automate part of the WordPress-side collection and add
deactivation-reason feedback — valuable for both KPIs and product decisions.

## Prioritisation matrices

Once objectives and data are in front of you, rank what to do next.

**Impact vs Effort (default, use this first):** plot each objective on
high/low impact × high/low effort. Do the **high-impact / low-effort** items
first; schedule high-impact/high-effort; drop or defer low-impact. This is the
fastest way to a sane backlog and fits a one-person operation.

**Eisenhower (urgent × important):** useful when time-sensitivity matters
(launch windows, version compatibility deadlines). Do urgent+important now,
schedule important/not-urgent, delegate/automate urgent/not-important, drop the
rest.

Don't treat either matrix as the only input — combine with the KPI data so
prioritisation is evidence-led, not vibes-led.

## The loop

Define vision & goals → set objectives & KPIs → measure data consistently →
prioritise by impact/effort → execute the top items → re-measure → re-prioritise.
The point is consistency: a small amount done every cycle beats sporadic bursts.
